# UI Флоу — Конфигуратор Ралюма
> Создан: 2026-02-25. Финальная версия флоу для разработки скелета.

---

## Страницы и роутинг

```
/login                    ← если не авторизован, всё редиректит сюда
/projects                 ← главная после входа
/projects/:id             ← редактор конкретного проекта
/admin                    ← только для admin/superadmin
```

---

## 1. `/login` — Вход

```
Страница: по центру карточка
  - Логин (input)
  - Пароль (input + показать/скрыть)
  - Кнопка "Войти"
  - Ошибка под кнопкой если неверно

После входа → редирект на /projects
```

---

## 2. `/projects` — Список проектов

```
Шапка (top navbar):
  - Логотип/название слева
  - Справа: имя пользователя, кнопка "Администрирование" (если admin), кнопка "Выйти"

Тело страницы:
  - Строка поиска (широкая) + фильтр "Все системы" (dropdown)
  - Кнопка "+ Новый проект" (правый верхний угол)
  - Таблица проектов: Номер | Заказчик | Система | Дата | [✏ открыть] [📋 копировать] [🗑 удалить]

Клик "+ Новый проект":
  → Открывается МОДАЛЬНОЕ ОКНО поверх страницы
  → В модале: поле "Номер проекта", выбор заказчика (radio), выбор системы (4 таба)
  → Кнопка "Создать" → создаёт проект в БД → редирект на /projects/:id

Клик ✏:
  → Редирект на /projects/:id

Клик 🗑:
  → Модал подтверждения "Удалить проект №2041?" → [Отмена] [Удалить]
```

---

## 3. `/projects/:id` — Редактор проекта

```
Шапка (breadcrumb):
  [← Проекты]  Проект № 2041 · КРОКНА · СЛАЙД        [Сохранить и выйти]

Макет: два столбца

ЛЕВЫЙ столбец (фиксированный, ~280px):
  ─ СЕКЦИИ ─────────────────
  [+ Добавить секцию]

  [карточка] Секция 1
             СЛАЙД 3р · 3 пан
             2070 × 2400
             [✏] [🗑]

  [карточка] Секция 2
             СЛАЙД 3р · 2 пан
             1800 × 2400
             [✏] [🗑]

  ─ ДОКУМЕНТЫ ───────────────
  [📄 Производственный лист]
  [📋 Накладная]
  [🪟 Заказ стекла]
  [🎨 Заявка в покраску]

ПРАВЫЙ столбец (основной):
  Показывает форму активной секции
  (кликнул на секцию в левом → форма открылась справа)

  Если ни одна не выбрана:
    Пустое состояние: "Выберите секцию или добавьте новую"
```

## 4. Форма секции (правый столбец)

Зависит от типа системы проекта:

### СЛАЙД → SlideForm (3 таба внутри формы)

```
Таб 1 "Основное":
  Секция №, Кол-во, Ширина, Высота, Стекло, Окрашивание, Цвет RAL, Углы

Таб 2 "Система":
  Рельсы (3р/5р), Порог, 1я панель (лев/пр), Кол-во панелей,
  Не используется полоса (появляется если панелей < макс),
  Межстекольный профиль,
  Профиль слева [чекбоксы], Профиль справа [чекбоксы]

Таб 3 "Фурнитура":
  Ручки и замки (варианты зависят от профиля выбранного в табе 2)
  Защёлки в пол, Отступ под ручку

Внизу формы (всегда видно):
  ┌──────────────────────────────┐
  │  SVG-схема: вид сверху       │
  │  панели обновляются live     │
  └──────────────────────────────┘
  [Сохранить секцию]  [Производственный лист]
```

### КНИЖКА → KnizhkaForm

```
Параметры секции + выбор системы (B25/B16/B17/C16/C17)
Справа от формы: список компонентов (обновляется мгновенно)
```

### ЛИФТ / ЦС

```
Заглушка "В разработке"
```

## 5. Умные зависимости в форме СЛАЙД

```
Выбрал "3-рельсовая" И панелей < 3
  → показать "Не используется полоса: Внешняя | Внутренняя"

Выбрал "5-рельсовая" И панелей < 5
  → показать то же поле

Выбрал профиль-замок 1081 + ручка-профиль 112
  → показать выбор замка: RS3018 | RS3019 | Без замка
  → скрыть выбор ручек

Выбрал п-профиль + RS1002
  → показать выбор ручки: Без (глух.) | Без (подвиж.) | RS3014 | RS3017 | Скоба
  → скрыть выбор замка

Выбрал п-профиль + ручка-профиль 112
  → нет ни ручек ни замков

Окрашивание = RAL стандарт или RAL нестандарт
  → показать поле "Цвет RAL"

Галочка "Угловое соединение"
  → показать "Внешняя ширина секции, мм"
```

## 6. Кнопки документов (левый столбец)

```
Клик любой кнопки документа:
```

## 7. `/admin` — Панель администратора

```
Доступ: только role = admin или superadmin
Если обычный user → редирект на /projects

Шапка: та же navbar + "← Проекты"

Тело:
  [🔍 Поиск пользователя...]    [+ Создать пользователя]

  Таблица: Логин | Имя | Роль | Активен | Действия
  Действия: [✏ изменить] [🔑 сбросить пароль]

Клик "+ Создать пользователя":
  → Модал: Логин, Имя, Пароль (+ кнопка "сгенерировать"), Роль, Заказчик, Активен
  → [Сохранить]

Клик 🔑:
  → Модал: показывает новый сгенерированный пароль (один раз)
  → "Скопируйте пароль, он больше не будет показан"
```

## 8. Схема переходов между страницами

```
[/login]
   │ успешный вход
   ▼
[/projects] ◄──────────────────────────────┐
   │                                        │
   ├── клик "+ Новый проект"               │
   │      └── модал → создать → /projects/:id
   │                                        │
   ├── клик ✏ на проекте ──► [/projects/:id]
   │                              │
   │                        клик секции
   │                              │
   │                         форма справа
   │                              │
   │                        "Сохранить и выйти"
   │                              └──────────┘
   │
   └── кнопка "Администрирование" → [/admin]
                                        │
                                  "← Проекты"
                                        └──► [/projects]
```

## 9. Принципы UX

1. **Не более одного уровня вложенности** — всё через модалы или боковую панель
2. **Мгновенный пересчёт** — SVG-схема и компоненты обновляются при каждом изменении
3. **Умные зависимости** — поля появляются/скрываются автоматически
4. **Автономер** — Секция № присваивается автоматически, можно скорректировать
5. **Inline валидация** — ошибки показываем сразу, не при сохранении
6. **Без page reload** — всё SPA, переходы мгновенные

## 10. Роли и доступ

```
superadmin → всё + управление admin
admin      → управление users + все проекты
user       → только свои проекты
```
