# Спецификация клона AppGlass

> Документ: план создания собственного конфигуратора безрамного остекления.
> Цель: превзойти оригинал AppGlass по удобству, скорости и функциональности.

---

## 1. Что делает оригинал AppGlass

### Функционал
- Создание проекта (название, система, толщина стекла)
- Конфигурация секций (длина, высота, кол-во панелей, тип/расположение дверей, углы)
- Автоматический расчёт:
  - Размеры стёкол (ширина, высота, тип панели)
  - Нарезка профилей (RS, длина, углы)
  - Перечень комплектующих (27+ артикулов)
- Генерация PDF-отчёта (8-10 страниц)
- Хранение проектов в облаке (JSF + БД на сервере)

### Недостатки оригинала
- Медленный UI (JSF/PrimeFaces, полные page reload на каждое действие)
- Нет оффлайн-режима
- Нет API — только браузер
- Нет истории изменений
- Нет экспорта в Excel/CSV
- Одна система (CORTINA_STATUS) — нет поддержки В16/В17
- Нет калькулятора для клиента (только для производства)
- Нет мобильного UI

---

## 2. Что делаем лучше

| Аспект | AppGlass | Наш клон |
|--------|---------|---------|
| UI скорость | Медленный (JSF, page reload) | Мгновенный (SPA, React/Vue) |
| Оффлайн | Нет | Да (PWA / localStorage) |
| API | Нет | REST API, можно интегрировать |
| Системы | 1 (B25) | Все: B25/B16/B17/C16/C17 |
| Экспорт | Только PDF | PDF + Excel + CSV |
| Расчёт стекла | В PDF | В UI сразу |
| Верификация | Нет | Встроенная (сравнение с 164 реальными проектами) |
| История | Нет | Git-подобная история версий проекта |
| Мобильный | Нет | Адаптивный дизайн |
| Цены | Нет | Можно добавить прайс-лист |

## 3. Стек технологий

### Backend (рекомендуемый вариант)
```
Python + FastAPI
  - Быстрый async API
  - Pydantic для валидации входных данных
  - Уже есть engine/comp_calc.py (расчётный движок)
  - Генерация PDF: ReportLab или WeasyPrint

SQLite (MVP) → PostgreSQL (production)
  - Хранение проектов
  - Пользователи и аутентификация
```

### Frontend
```
React + TypeScript (или Vue 3)
  - Компонент конфигуратора секций
  - Интерактивный чертёж (SVG или Canvas)
  - Таблица комплектующих в реальном времени
  - Tailwind CSS для стилей
```

### Альтернатива (monолитный стек)
```
FastAPI + Jinja2 + HTMX
  - Проще деплой
  - Без сборщика (webpack/vite)
  - Хорошо для MVP
```

## 4. Модель данных

### Project
```python
class Project:
    id: UUID
    name: str                    # "В25-5-4277"
    system_type: SystemType      # 'B25' | 'B17' | 'C17' | 'B16' | 'C16'
    glass_thickness: int         # 8 | 10
    created_at: datetime
    updated_at: datetime
    sections: List[Section]
```

### Section
```python
class Section:
    id: UUID
    project_id: UUID
    order: int                   # порядковый номер (1, 2, 3...)
    length: float                # мм
    height: float                # мм
    n_panels: int                # кол-во панелей
    n_doors: int                 # кол-во дверей
    left_angle: float            # º (0 если нет)
    right_angle: float           # º (0 если нет)
    door_side: DoorSide          # 'left' | 'right' | 'both' | None
    door_type: DoorType          # 1-7 (тип фурнитуры)
    door_opening: DoorOpening    # 1-4 (направление открытия)
    compensator: CompensatorType # 0-9
    is_biparting: bool
    opening_out: bool
```

### CalculationResult (кешируется, пересчитывается при изменении секций)
```python
class CalculationResult:
    project_id: UUID
    glasses: List[GlassSpec]     # размеры стёкол
    profiles: List[ProfileCut]   # нарезка профилей
    components: Dict[str, float] # артикул → количество
    warnings: List[str]
    calculated_at: datetime
```

## 5. API Endpoints

### Проекты
```
GET    /api/projects              — список проектов
POST   /api/projects              — создать проект
GET    /api/projects/{id}         — получить проект
PUT    /api/projects/{id}         — обновить проект
DELETE /api/projects/{id}         — удалить проект
POST   /api/projects/{id}/copy    — скопировать проект
```

### Секции
```
GET    /api/projects/{id}/sections            — список секций
POST   /api/projects/{id}/sections            — добавить секцию
PUT    /api/projects/{id}/sections/{sec_id}   — изменить секцию
DELETE /api/projects/{id}/sections/{sec_id}   — удалить секцию
POST   /api/projects/{id}/sections/reorder    — изменить порядок
```

### Расчёт
```
GET    /api/projects/{id}/calculate           — пересчитать (кешируется)
POST   /api/projects/{id}/calculate/force     — принудительный пересчёт
```

### Экспорт
```
GET    /api/projects/{id}/export/pdf          — PDF отчёт
GET    /api/projects/{id}/export/excel        — Excel спецификация
GET    /api/projects/{id}/export/csv          — CSV компоненты
```

## 7. Алгоритм расчёта

Движок уже реализован в `engine/comp_calc.py`.

### Порядок расчёта:
```
1. Validate sections (длина > 0, n_panels ≥ n_doors, etc.)
2. Aggregate: N_total, N_door, N_fix, N_sec, L_total
3. Determine flags: has_angular, has_biparting, system_type
4. calc_components(sections, system_type) → Components
5. glass_calc(sections) → List[GlassSpec]        [engine/glass_calc.py]
6. profile_calc(sections) → List[ProfileCut]     [нужно реализовать]
7. Cache result, return to API
```

### Системы и их особенности:
```python
# В движке engine/comp_calc.py
# system_type → _OLD16_FS_SYSTEMS: 'B16', 'C16' → FS3110 вместо FD3110, S0057=0
# Для С17: FD3110=N_sec, RS3110=N_fix-N_sec (нужно добавить)
```

**Что нужно доработать в движке:**
- [ ] С17: RS = N_fix − N_sec, FD = N_sec
- [ ] С17: RS (biparting) = (N_total − N_sec) / 2
- [ ] D0012 второе значение для угловых проектов
- [ ] P0003/P0004 для door_type=3,5,6 → N_total вместо N_fix
- [ ] profile_calc.py — нарезка профилей (пока нет)

## 10. Приоритеты реализации (порядок)

### MVP (минимально работающий продукт)
1. **backend**: FastAPI + SQLite + engine/comp_calc.py
2. **API**: CRUD проектов/секций + расчёт компонентов
3. **frontend**: форма создания проекта, список секций, таблица компонентов
4. **PDF**: базовый отчёт (страница компонентов)

### V1 (полноценный)
5. Все системы (B25/B16/B17/C16/C17)
6. Расчёт стёкол + нарезка профилей
7. Полный PDF (все 6 страниц)
8. Excel экспорт
9. Интерактивный чертёж (SVG)
10. Верификация на 164 проектах

### V2 (сверх AppGlass)
11. Прайс-лист и расчёт стоимости
12. История изменений проекта
13. Мультипользовательский доступ
14. Мобильный UI (PWA)
15. Публичный API для интеграции
