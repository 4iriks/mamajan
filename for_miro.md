# Ралюма — текущее состояние проекта
> Проверено по реальному коду, не по документации. Дата: 2026-03-26

---

## Что реально работает сейчас

### Авторизация
- JWT-токен в localStorage, `/api/auth/login` + `/api/auth/me`
- При перезагрузке страницы сессия восстанавливается через `/me`
- Три роли: `user`, `admin`, `superadmin`
- Protected routes на фронтенде

### Проекты (список)
- Создание, редактирование (переименование inline), удаление, копирование
- Поиск по номеру и заказчику
- Два таба: **Текущие** / **Архив** (фильтр по `status = 'Архив'`)
- Датасписок заказчиков (подсказки из существующих проектов)
- Маска номера проекта (Х00-0-0000)
- Цветные бейджи статусов (производство, стекло, покраска)
- Skeleton-загрузка строк

### Редактор секции — структура
- Список секций в сайдбаре
- Добавление новой секции через пикер системы (аккордеон)
- Нумерация секций
- Кнопка «← К проекту»
- Guard несохранённых изменений (beforeunload + предупреждение)
- Сохранение через кнопку и Ctrl+S
- Хлебные крошки (проект → секция)
- Примечания к секции

### Редактор секции — формы
Реализованы табы для **5 систем**:
| Система | Таб | Что есть в форме |
|---------|-----|------------------|
| СЛАЙД | `SlideSystemTab` | рельс, порог, фурнитура, взаимоисключение RS112/RS1002, автовыбор RS1081→RS1082, профили боковые, ручка, замок |
| КНИЖКА | `BookSystemTab` | подтип, ручки, замки, система |
| ЛИФТ | `LiftSystemTab` | подтип, ручки |
| ЦС | `CsShapeTab` | форма, ширина |
| КОМПЛЕКТАЦИЯ | `DoorSystemTab` | подтип двери, сторона, открывание |

**MainTab** (общий для всех систем): ширина, высота, кол-во панелей, количество секций, тип стекла, цвет/RAL, покраска, угол левый/правый.

### Визуализация (SVG)
- **Вид сверху** (`SlideSchemeSVG`): схема СЛАЙД с профилями УЛИЦА/ПОМЕЩЕНИЕ, размерные линии, подписи артикулов
- **Вид из помещения** (`SlideRoomViewSVG`): фронтальная проекция с размерными линиями
- Схемы sticky на десктопе, скрыты на мобильном (xl:hidden)
- Только для системы СЛАЙД (другие — заглушки)

### Статусы производства (на уровне проекта)
- Статус производства (выпадашка)
- Стекло: статус, накладная, дата готовности
- Покраска: статус, дата отгрузки, дата получения
- Заказы: список позиций с qty (JSON в БД)
- 1 или 2 стадии производства (выбор при создании проекта)

### Админка
- CRUD пользователей (создание, редактирование, удаление, сброс пароля)
- Bulk-операции (смена роли группе)
- Только для `admin` / `superadmin`
- Superadmin может создавать других superadmin

### Инфраструктура
- Docker Compose: FastAPI backend + React/Vite frontend + Caddy (auto-SSL)
- Деплой на VPS `89.111.142.17` → `raluma.tech` (HTTPS)
- SQLite + SQLAlchemy
- Ручные ALTER TABLE миграции при старте (`migrations.py`)
- GitHub Actions CI (ESLint + pytest)
- Pre-commit хуки

### База данных — ключевые поля Section (все в БД, все на фронте)
`width, height, panels, quantity, glass_type, painting_type, ral_color, threshold, rails, inter_glass_profile, profile_left/right, lock, handle, floor_latches_left/right, handle_offset, corner_left/right, external_width` — и ещё ~20 полей для книжки/двери/ЦС.
